How Wrong-Answer Recycling Works

During a session: When you answer a question wrong, it goes back into the queue and resurfaces later. The session only ends when all questions have been answered correctly.

Session length varies: A perfect run = 10 questions. A session with 3 mistakes = 13 questions (10 original + 3 retries).

No failure state: Every session ends with 100% — the question is how many retries it took. "Perfect!" badge for zero retries.

Session History Shows Results, Not Scores

Perfect run: ⭐ with green "Perfect!" badge — no mistakes

With retries: ✅ with orange "3 retries" badge — shows total questions (13 = 10 original + 3 retries)

Topics shown after the fact — user sees which topics the algorithm focused on, but only after completing the session

XP scales: More XP for fewer retries (perfect = max XP)

Technical: Retry Queue in Session

Modify the session flow in the store:

1. When answerQuestion is called with correct: false, push the question back into session.questions at a random position 2-4 slots later

2. Track session.retryCount to count total retries

3. Session completes when currentIndex >= questions.length (array grows as retries are added)

4. totalQuestions in progress bar = original count (10), but the bar doesn't go past 100% — retries extend the session
