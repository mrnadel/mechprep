1. Mistake-Specific Feedback DONE
When a user picks the wrong option, Octokeen shows the correct answer but never explains WHY their specific choice was wrong.
CRITICAL Learning Content
▼

Code evidence: CourseQuestion type (types.ts lines 70-115) has only a single explanation: string field. Options are string[] with no per-option metadata. Grep for distractorExplanation|optionExplanation|optionFeedback returns zero matches. QuestionCard.tsx shows identical explanation text regardless of which wrong option was selected.

Duolingo
  • Shows why your specific answer was wrong
  • "Explain My Answer" gives detailed breakdowns
  • Feedback is contextual to the mistake
Octokeen Today
  • Shows correct answer in green, wrong in red
  • Single explanation field explains the concept generically
  • Same feedback regardless of which wrong option chosen

Implemented

  • Added distractorExplanations: Record<number, string> to CourseQuestion type
  • QuestionCard, ScenarioCard, PickTheBestCard pass selectedOriginalIndex through onAnswer
  • LessonView renders staggered distractor callout (0.15s) then explanation (0.35s) with graceful fallback
  • selectedIndex logged in mastery events for distractor analytics
  • ~11,000 distractor explanations written across ~130 content files (all 4 courses)
  • Validation test at src/__tests__/data/distractor-coverage.test.ts
🕐
2. Spaced Repetition in Lesson Flow DONE
Mastery decay scoring exists but is never used to resurface questions in course lessons. The infrastructure is there — it just isn't wired up.
CRITICAL Learning Algorithm
▼

Code evidence: mastery.ts has HALF_LIFE_DAYS = 14 with exponential decay formula. But useCourseStore.ts lines 279-296 selects lesson questions purely from the lesson's own question pool — mastery scores are never referenced. No "Review" practice mode exists. No "Needs Review" indicators on course map. Practice algorithm (practice-algorithm.ts) uses mastery for practice sessions only.

Duolingo
  • Half-Life Regression predicts forgetting per item
  • Review sessions surface decaying knowledge
  • "Practice" button on completed lessons
  • Review items interleaved into new lessons
Octokeen Today (Foundation Exists)
  • Mastery decay with 14-day half-life — good math!
  • Practice algorithm weights weak topics higher
  • But course lessons don't use mastery scores at all
  • No review mode, no "needs review" on map

Implemented

  • Created src/lib/review-engine.ts with per-question mastery decay detection (threshold < 50)
  • useCourseStore.startLesson() interleaves 1-2 review questions from earlier units
  • LessonView resolves review questions across unit boundaries
  • New /practice/review/ page — dedicated Review mode with 10-question sessions
  • Pulsing amber review badge on LessonNode for completed lessons with decayed mastery
  • CourseMap computes review status from mastery events and passes to nodes
